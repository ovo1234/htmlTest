사이버 보안 : 해킹, DDos, 디페이스, 피싱, 랜섬웨어 등 사이버 침해로부터 인터넷을 안전하게 사용하도록 보호하는 활동
 - 사이버 침해사고 유형
    ㄴ 해킹 : 정당한 접근권한 없이 정보통신시스템에 침입 / 악성 코드를 유포, 감염시켜 원격조정 및 공격 시도
    (패스워드 크래킹 방법) 사전공격 : 사전에 나오는 단어를 패스워드에 대입
                        무작위 대입 공격 : 특정한 암호를 풀기 위해 가능한 모든 값을 대입
    (S/W 취약점) PC 어플리케이션 퍼징도구 : 프로그램에 오류를 다양한 값을 입력하여, 프로그램의 반응을 조사(퍼징기법)하는 블랙박스 방식
                웹 취약점 탐지 도구 : 웹 서비스는 소스코드를 확보할 수 있으므로, 소스코드 상에서 취약성을 탐지하는 화이트박스 방식과 블랙박스 방식을 같이 사용
                침투시험을 통한 취약성 검증 도구 : 알려진 취약점 정보를 바탕으로, 점검 대상 시스템에 직접 침투하여 취약점을 검증하는 방식
    ㄴ 분산서비스거부(DDos) : 악성코드에 감염된 다수의 PC를 이용, 대량의 트래픽을 특정 시스템에 전송하여 정상적인 서비스를 방해하는 공격
    ㄴ 디베이스(Deface) : 홈페이지의 화면을 변조시켜 다른 이미지가 보이도록 하는 침해사고 / 특정 기업이나 조직의 이미지를 실추시키려고 함.
    ㄴ 피싱 : 불특정 다수에게 메일 등을 통해 위장된 홈페이지에 정보를 입력하여 개인정보, 금융정보등을 빼내는 기법
    ㄴ 스팸 : 수신자 의사에 반하여 정보통신망을 통해 일방적으로 전송되는 영리목적의 광고성 정보 / 악성코드에 감염된 PC나 대포폰등을 통해 개인정보로 스팸 발송
    ㄴ 랜섬웨어 : 사용자의 동의 없이 컴퓨터를 암호화시켜 금전적인 행위를 하는 악성 프로그램
    ㄴ 딥페이크 : 인공지능을 기반으로 인간 이미지 합성 기술

침해사고 동향 : APT 공격 증가
    => 지능형 지속 위협(APT)을 통한 공격 증가
    => APT 공격의 의미 / 모든 수단과 방법을 동원하여 정해진 목표가 달성될 때가지 지속적으로 대상에 위협을 가하는 행위
    침투 -> 탐색 -> 수집 -> 유출
    특징 : 경제적, 정치적 및 전략적 이득을 위한 공격 / 공격 대상의 주변 환경을 점령하여 관리하고, 필요 시에 다시 방문할 수 있도록 구축 / 상당히 오랫동안 공격이 진행됨에도 인지하기 어려움
    단계 :
    1. 초기 정찰 : 공격 목표에 대한 정보 수집 단계
    2. 초기 침입 : 조직 내부 넷으로 초기 악성코드를 유입시키고 침투하는 단계
    3. 거점 마련 : 다양한 기능을 수행하는 백도어를 통한 공격자와의 연결 생성 및 추가 공격을 위한 공격 도구를 다운로드함
    4. 권한 상승 : 시스템의 관리자 권한 필요
    5. 내부 정찰 : 공격 도구를 활용하여 공격 대상 내부 시스템/넷 정보를 수집
    6. 내부 침투 : 내부 정찰을 통해 수집한 정보를 이용하여 내부 넷상에 있는 다른 시스템을 추가 공격하고 제어 권한을 탈취
    7. 지속성 유지 : 백도어를 통한 표적 시스템에 대한 연결을 지속적으로 유지하는 단계
    8. 목표 달성 : 정보 유출, 시스템 파괴 등 최종 공격 목표를 달성

    1. 스피어 피싱 : 특정인을 표적으로 이메일 발송하여 악성 웹사이트, 악성 첨부 파일로 악성 코드에 감염시킴
    2. 워터링 홀 : 표적 공격의 일종으로 매복하고 있다가 악성코드를 심어 감염시킴

악성코드와 Anti-Virus
    악성코드 : 제작자가 의도적으로 사용자에게 피해를 주고자 만든 모든 악의적인 목적을 가진 프로그램 및 매크로, 스크립트 등 컴퓨터 상에서 작동하는 모든 실행 가능한 형태의 소프트웨어
    - 바이러스 : 사용자 몰래 변형해 복사하는 프로그램(복제, 감염) / 스스로 전파되지 않음
    - 웜 : 인터넷 또는 넷를 통해 컴퓨터에서 컴퓨터로 전파되는 악성 프로그램 / 스스로 전파됨 
    - 트로이목마 : 직접적인 피해를 주지 않지만 사용자의 컴퓨터를 조종하는 프로그램 , 정상적인 파일에 포함되어 함께 설치, 자기 자신을 다른 파일에 복사하지 않음.

    Anti-Virus : 악성코드를 탐지하고 치료하는 컴퓨터 프로그램 
    동작원리 : 바이러스 데이터베이스의 정의와 일치하는 바이러스를 확인하기 위해 파일의 내용을 확인
        감염으로 표시될 가능성이 있는 컴퓨터 프로그램에서 의심이 가는 행동을 하는지 확인

침해사고 원인 : 악성코드에 감염된 좀비 PC 악용
    좀비 PC : 해커에 의해 악성코드에 감염되어 공격자의 명령에 따라 PC 사용자 의도와 관계없이 각종 악의적인 행위를 수행

    최초의 컴퓨터 연동망 ARPA (연구 기관과 국방 관련 사업체 등 관련 기관 사이의 정보 공유를 지원하기 위한 프로젝트로 컴퓨터 연동망 개발)

    애플 컴퓨터의 탄생

    정보 권리 논쟁의 시작
    자유 소프트웨어 대한 4가지 자유
        ㄴ 자유 0 : 프로그램을 어떠한 목적을 위해서도 실행할 수 있는 자유
        ㄴ 자유 1 : 프로그램의 작동 원리를 연구하고 이를 자신의 필요에 맞게 변경시킬 수 있는 자유
            -> 이를 위해 소스 코드에 대한 접근이 선행되어야 함.
        ㄴ 자유 2 : 이웃을 돕기 위해서 프로그램을 복제하고 배포할 수 있는 자유
        ㄴ 자유 3 : 프로그램을 향상시키고 이를 공동체 전체의 이익을 위해서 다시 환원시킬 수 있는 자유
            -> 이를 위해 소스 코드에 대한 접근이 선행되어야 함.

2. 정보 보안의 이해
 1) 보안의 3대 요소
    기밀성 : 인가된 사용자만 정보 자산에 접근할 수 있는 것, 방화벽, 암호, 패스워드 등
    무결성 : 적절한 권한을 가진 사용자가 인가한 방법으로만 정보를 변경할 수 있도록 하는 것
    가용성 : 필요한 시점에 정보 자산에 대한 접근이 가능하도록 하는 것
 2) 보안 전문가의 자격 요건
    사이버 테러형 범죄 : 정보통신망 자체를 공격대상으로 함 / 전자기적 침해 장비를 이용해 컴퓨터 시스템과 정보통신망을 공격하는 행위
    일반 사이버 범죄 : 사이버 공간을 이용한 일반적인 불법 행위 / 사이버 도박, 사이버 스토킹 등
    기본 소양(윤리 의식) : 해킹은 절대 하면 안 된다는 사실을 인지시키기위한 여러 가지 윤리 강령이 발표됨
    지식 소양(다양한 분야의 전문성) : 보안 전문가가 전문성을 갖춰야 할 분야에는 운영체제, 네트워크, 프로그래밍, 서버 등 
    정보통신 윤리강령

 2) 데이터베이스의 접근 제어

        DDL : 데이터베이스를 처음 생성하고 개발할 때 주로 사용하고 
            - CREATE : 데이터베이스 객체 생성
            - DROP : 데이터베이스 객체 삭제
            - ALTER : 기존의 데이터베이스 객체를 다시 정의
        DML : 질의문으로 데이터의 검색과 수정을 처리

            - UPDATE : 기존 데이터베이스 객체에 있는 데이터 수정
            - DELETE : 데이터베이스 객체 있는 데이터 삭제
        DCL : 권한 관리를 위한 질의문
            - GRANT : 데이터베이스 객체에 권한을 부여
            - DENY : 사용자에게 해당 권한을 금지
            - REVOKE : 이미 부여된 데이터베이스 객체의 권한을 취소

3장 네트워크 보안

1. 네트워크의 기초
 1) 네트워크의 이해
    OSI 7계층
    - 7계층 : 응용 프로그램 계층 / 응용 프로세스와 직접 관계하여 일반적인 응용 서비스를 수행
    - 6계층 : 표현 계층 / 코드 간 번역을 담당
    - 5계층 : 세션 계층 / 양 끝단의 응용 프로세스가 통신을 관리하는 방법을 제공
    - 4계층 : 전송 계층 / 포트를 이용해 시스템 내의 응용 프로그램 간 통신을 가능하게 함
    - 3계층 : 네트워크 계층 / IP 주소를 통해 원거리 시스템 간 통신이 가능하게 함
    - 2계층 : 데이터 링크 계층 / 물리 장치의 고유 주소인 MAC을 이용한 로컬 통신이 가능하게 함
    - 1계층 : 물리 계층 / 신호를 전달하기 위한 물리적인 연결
 2) 물리계층
    인터넷 이용 시 랜, 전화선, 동축 케이블 또는 광 케이블 등 시스템 간의 물리적인 연결매체
    케이블 선의 분류
        ㄴ UTP : 두 선간의 전자기 윧를 줄이기 위하여 절연의 구리선이 꼬여있음.
        ㄴ FTP : 알루미늄 은박이 4가닥의 선을 감싸고 있음
        ㄴ STP : 연선으로 된 케이블 겉에 외부 피복 또는 차폐재가 추가되어 외부의 노이즈를 차단하거나 전기적 신호의 간섭에 탁월함.
 3) 데이터 링크 계층 
    두 포인트 간의 신뢰성 있는 전송을 보장 (LAN 카드 => 물리주소 정의/Data를 frame에 전송)
        ㄴ Mac 주소는 총 12개의 16진수 숫자로 구성
            ㄴOUI : 네트워크 카드를 만든 회사를 뜻함
            ㄴHost Identifier : 각 회사에서 생성되는 네트워크 카드에 임의로 붙이는 일종의 시리얼의 의미
    데이터 링크 계층의 대표적인 네트워크 장비는 스위치, MAC계층에서 동작하는 프로토콜은 이더넷(1계층과 2계층에서의 동작)
    패킷의 구조 : 3계층까지의 패킷 정보 / 2계층의 패킷 정보(출발지 MAC 주소 / 목적지 MAC 주소)
    스위치의 동작 원리 : 
        
 4) 네트워크 계층 : 논리주소 표현(IP주소) / 여러 개의 노드를 거칠 때마다 경로를 찾아주는 역할을 하는 계층
    ㄴ 다양한 길이의 데이터를 네트워크를 통해 전달하며 그 과정에서 라우팅, 흐름 제어, 세그멘테이션, 오류 제어 등을 수행
    IP 주소 : 8비트의 수 4개로 구성됨(32자리의 2진수로 8자리마다 점을 찍어 구분)
        ㄴ A 클래스 : 첫 번째 자리가 네트워크 주소, 나머지 세 자리가 호스트 주소
        ㄴ B 클래스 : 두 번째 자리까지가 네트워크 주소, 나머지 두 자리가 호스트 주소
        ㄴ C 클래스 : 세 번째 자리까지가 네트워크 주소, 나머지 한 자리가 호스트 주소

 5) 전송 계층 : 상위 계층이 데이터 전달의 유효성이나 효율성을 신경 쓰지 않도록 해줌
    계층의 대표적인 프로토콜 : TCP
    MAC 주소가 네트워크 카드의 고유 식별자이고 IP가 시스템의 주소라면, 포트는 시스템에 도착한 후 패킷이 찾아갈 응용 프로그램과 통하는 통로 번호라 생각함.
    시스템에서 구동되는 응용 프로그램들은 네트워킹을 하기 위해 자신에게 해당되는 패킷을 식별할 필요가 있으며, 사용하는 것이 포트임.
    주요 포트와 서비스 : 0~1023번 포트까지 1024개를 Well Known 포트라고 함.
    : 출발지 포트는 운영체제나 응용 프로그램마다 조금씩 다름
    : 보통 1025번부터 65535번 사이의 포트 중에서 사용하지 않는 임의의 포트를 응용 프로그램 별로 할당하여 사용함.
    세션연결 
        ㄴ 1단계 : 클라이언트는 포트가 닫힌 상태이며 서버는 항상 Listen 상태임.
        ㄴ 2단계 : 처음 클라이언트가 통신을 하고자 하면, 임의의 포트 번호가 프로그램에 할당 -> 서버에 연결하고 싶다는 의미로 SYN Sent 상태가 됨.
        ㄴ 3단계 : 클라이언트의 연결 요청을 받은 서버는 SYN Received 상태가 되고, 클라이언트에게 연결을 해도 좋다는 의미로 패킷을 보냄
        ㄴ 4단계 : 클라이언트는 연결을 요청한 것에 대한 서버의 응답을 확인했다는 표시로 ACK 패킷을 서버에 보냄.
    TCP 세션의 종료
        ㄴ 1단계 : 통신을 하는 중에는 클라이언트와 서버 모두 Established 상태
        ㄴ 2단계 : 통신을 끊고자 하는 클라이언트가 서버에 FIN 패킷을 보내는데, 이때 클라이언트는 Close Wait 상태가 됨.
        ㄴ 3단계 : 서버는 연결 종료 요청을 확인하고 ACK 패킷을 보내면 , 서버도 FIN 패킷을 보내고 Close Wait 상태가 됨.
        ㄴ 4단계 : 클라이언트는 연결 종료를 요청한 것에 대한 서버의 응답을 확인했다는 의미로 ACK 패킷을 서버에 보냄.
    TCP : 연결 지향형 프로토콜로서 수신측이 데이터를 흘려버리지 않게 데이터 흐름 제어와 전송 중 에러가 발생할 경우 자동으로 재전송하는 에러 제어 기능을 통해 데이터의 확실한 전송을 보장
    UDP : 데이터의 신뢰성 전송을 보장 X / 전송 경로 확립을 위한 번잡함을 생략하고 시간을 절약할 수 있어 더 효과적임.

 6) 세션 계층 , 표현 계층 , 응용 프로그램 계층
    세션 계층 : 기본적으로 두 시스템 간의 통신 중 동기화를 유지하고 데이터 교환을 관리함
                전송계츠ㅓㅇ이 종단 간에 논리적인 설정을 담당하면 정보 교환을 효과적으로 할 수 있게 추가 서비스함.
    표현 계층 : 데이터의 구조를 하나의 통일된 형식으로 표현함
                데이터의 압축과 암호화 기능을 수행함
    응용 프로그램 계층 : 사용자나 응용 프로그램 사이에 데이터의 교환이 가능하게 하는 계층

2. 네트워크 장비에 대한 이해
 1) 물리 계층 : 전화선의 연결 커넥터 (RJ-11) , 랜 케이블의 연결 커넥터를 RJ-45 라고 함.
    리피터 : 네트워크 연장 장비 / 불분명해진 신호 세기를 다시 증가시키는 역할
    더미 허브 : 스위치의 옛날 형태 / 스위칭 허브라고 부름
