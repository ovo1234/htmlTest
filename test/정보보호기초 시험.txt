DDos 공격(분산서비스거부 공격) : 공격자가 악성코드에 감염된 다수의 PC를 이용하여 대량의 유해 트래픽을 전송하여 서비스를 방해하는 공격(공격자, 마스터, 에이전트, 공격대상으로 구성)
    ㄴ 공격자 : 공격을 주도하는 해커의 컴퓨터
    ㄴ 마스터 : 공격자에게 직접 명령을 받는 시스템으로 여러 대의 에이전트를 관리함
    ㄴ 핸들러 프로그램 : 마스터 시스템의 역할을 수행하는 프로그램
    ㄴ 에이전트 : 공격 대상에 직접 공격을 가하는 시스템
    ㄴ 데몬 프로그램 : 에이전트 시스템의 역할을 수행하는 프로그램
    
디페이스 / 피싱 / 파밍 / 스팸 / 랜섬웨어 / 딥페이크 (객관 구별)

    => APT 공격에서 초기 침입시 주로 사용되는 기법
        ㄴ 스피어 피싱 : 특정인을 표적으로 이메일 발송 -> 악성 코드에 감염시키는 방식
        ㄴ 워터링 홀 : 표적 공격의 일종으로 공격자는 주로 방문하는 웹사이트에 취약점을 이용하여 해당 사이트에 악성코드를 심어두고 공격 대상이 해당 사이트에 접근하면 악성코드에 감염시키는 방식
        ㄴ USB 메모리 스틱을 이용하는 방법 : 메모리를 감염 후 누군가 이를 가져가서 내부 pc에 연결할 때 감염되는 방식

데이터베이스의 권한관리
    질의문에 대한 권한 관리
        DDL : 데이터베이스를 처음 생성하고 개발할 때 주로 사용하고 
            - CREATE : 데이터베이스 객체 생성
            - DROP : 데이터베이스 객체 삭제
            - ALTER : 기존의 데이터베이스 객체를 다시 정의
        DML : 질의문으로 데이터의 검색과 수정을 처리
            - SELECT : 사용자가 테이블이나 뷰의 내용을 읽고 선택
            - INSERT : 데이터베이스 객체에 데이터 입력
            - UPDATE : 기존 데이터베이스 객체에 있는 데이터 수정
            - DELETE : 데이터베이스 객체 있는 데이터 삭제
        DCL : 권한 관리를 위한 질의문
            - GRANT : 데이터베이스 객체에 권한을 부여
            - DENY : 사용자에게 해당 권한을 금지
            - REVOKE : 이미 부여된 데이터베이스 객체의 권한을 취소

로그 관리 - 유닉스의 로그
    WTMP : 사용자 로그인과 로그아웃, 시스템 재부팅에 대한 정보를 확인 가능(var/log/wtmp에 위치) lost 명령어 사용
    UTMP : 현재 로그인 사용자 상태 정보를 담고 있는 로그파일(var/run/utmp) w, who, finger 명령어 사용
    BTMP : 실패한 로그인 정보를 담고 있는 로그파일(var/log/btmp) lastb 명령어 사용
    LAST LOG : 마지막으로 성공한 로그인 정보를 담고 있는 로그파일 (var/log/lostlog) lastlog 명령어 사용
